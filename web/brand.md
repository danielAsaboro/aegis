# Brand - AEGIS Web

Status: active

AEGIS is a policy-gated autonomous onchain agent built on top of the forked Zerion CLI.
The visual identity should feel like a premium execution cockpit: security-first, editorial,
technical, and judge-verifiable.

## Palette

- Ink: `#070707`
- Panel: `#12120f`
- Text: `#f6f0df`
- Muted text: `#a69c88`
- Signal gold: `#deb259`
- High gold: `#f2cf7a`
- Guard green: `#57f287`
- Zerion blue: `#4f7cff`
- Shield cyan: `#67e8f9`

Use gold for the primary brand signal, green only for policy/pass states, blue for Zerion
execution references, and cyan for shield or tool-call highlights. Avoid purple-dominant SaaS
gradients and plain white cards.

## Typography

- Display: Syne
- Body: DM Sans
- Mono: JetBrains Mono

Display type is bold and editorial. Body copy stays concise, factual, and grounded in the
repo's real capabilities.

## Voice

Lead with guardrails and execution proof. AEGIS can act, but it cannot bypass policy. Prefer
specific claims such as "forked Zerion CLI", "fail-closed", "human approval", and "Zerion API"
over generic AI-agent language.
