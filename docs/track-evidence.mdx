## 2. Policy and no-bypass proof

AEGIS is not an unconstrained wallet agent. Every value-moving path is scoped by policies and write-tool approval.

| Proof | Evidence |
| --- | --- |
| Fail-closed empty config | `engine/policies/engine.mjs` throws `MissingPolicyConfigError` when no policy is active. |
| Executor hard stop | `engine/execution/executor.mjs` throws when a trade lacks an approved `policyResult`. |
| Swap tool gate | `engine/agent/tools/swap.mjs` builds a proposal, runs policies, and only then calls `executeTrade()`. |
| Human approval | Telegram approval prompts live in `engine/bot/handlers/chat.mjs`; write tools declare approval requirements. |
| Unit coverage | `tests/unit/policies/no-bypass.test.mjs`, `tests/unit/agent/no-bypass.test.mjs`, `tests/unit/agent/tool-contract.test.mjs`. |

Quick local proof:

```bash
pnpm judge-trace
pnpm test:unit
```

## Disclosed limitations

- MagicBlock withdraw after a `private: true` deposit currently fails with `DelegationRecordInvalidAccountOwner` in the SDK flow described in `TRACKS.md`.
- MagicBlock private execution is verified for deposit and delegation, not for the full private transfer plus withdraw cycle.
- Local QVAC model performance depends on the installed model and CPU/GPU configuration. AEGIS extends the Telegram handler timeout for `qvac/*` models instead of pretending local CPU inference is instant.
- API-key billed LLM routes such as `openai/*` and `anthropic/*` are intentionally unsupported.
