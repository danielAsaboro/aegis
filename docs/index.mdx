AEGIS turns natural-language trading requests into real onchain actions without replacing the Zerion CLI underneath. The wallet, swap routing, and execution layer stay inside the forked Zerion stack. AEGIS adds the agent loop, policy engine, automation, and private execution path on top.

<Info>
  Frontier constraint: AEGIS remains a Zerion CLI fork. Real swaps still route through Zerion and value-moving actions stay policy-gated.
</Info>

## What AEGIS adds on top of Zerion

- A natural-language agent loop over Telegram, long-running runtime automation, and MCP.
- A fail-closed policy engine that runs before every value-moving action.
- Strategy automation for DCA, signals, portfolio actions, and consensus flows.
- A MagicBlock privacy path for shielded execution.
- Optional local-first AI through QVAC instead of API-key billed model providers.

## Core guarantees

<Check>
  All onchain actions continue to route through the forked Zerion CLI and the Zerion API path where required.
</Check>

<Check>
  There is no bypass flag for swaps, bridges, shield operations, or scheduled executions.
</Check>

<Check>
  The same policy engine protects Telegram-driven actions, automated strategies, and tool calls coming from external MCP hosts.
</Check>
